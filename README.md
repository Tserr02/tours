# Веб-приложение по продаже туров

Проект реализован по курсовой работе: Django-приложение для туристической фирмы с каталогом туров, поиском, заявками, личным кабинетом, избранным, отзывами, справочником стран и административной панелью.

## Возможности

- Главная страница с быстрым поиском туров.
- Каталог с фильтрами по стране, курорту, датам, длительности, питанию, звездам отеля и цене.
- Сортировка по цене, длительности и рейтингу отеля.
- Детальная карточка тура с галереей, описанием отеля, перелетом, трансфером, отзывами и похожими турами.
- Регистрация, вход, личный кабинет и редактирование профиля клиента.
- Предзаполнение заявки данными авторизованного пользователя.
- Добавление туров в избранное.
- Подписка на рассылку.
- Административная панель менеджера для управления странами, курортами, отелями, турами, заявками и отзывами.
- PostgreSQL в качестве основной базы данных.

## Запуск

Для запуска через Visual Studio Code можно использовать готовые задачи из файла [VS_CODE_SETUP.md](VS_CODE_SETUP.md).

1. Создайте виртуальное окружение и установите зависимости:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

2. Запустите PostgreSQL через Docker:

```powershell
docker compose up -d
```

3. При необходимости создайте файл `.env` по примеру `.env.example` или задайте переменные окружения. По умолчанию приложение использует:

```text
POSTGRES_DB=tours_db
POSTGRES_USER=tours_user
POSTGRES_PASSWORD=tours_password
POSTGRES_HOST=localhost
POSTGRES_PORT=5432
```

4. Примените миграции и заполните демо-данные:

```powershell
python manage.py migrate
python manage.py seed_demo
python manage.py createsuperuser
```

5. Запустите сайт:

```powershell
python manage.py runserver
```

После запуска:

- сайт: http://127.0.0.1:8000/
- админ-панель: http://127.0.0.1:8000/admin/

Проверка тестов:

```powershell
python manage.py test
```

## Структура

- `config/` - настройки Django-проекта.
- `tours/` - основное приложение: модели, формы, маршруты, представления, админка, миграции.
- `templates/` - HTML-шаблоны страниц.
- `static/css/styles.css` - стили интерфейса.
- `docker-compose.yml` - PostgreSQL для локального запуска.
